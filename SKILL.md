# 大师的AI小灶 · 脚本写作

## 写作流程

### 1. 明确输入

确认以下信息（缺失的主动询问）：
- **选题**：视频主题是什么
- **类型**：教程 / 评测 / 观点 / 广告（广告需确认品牌和卖点）
- **时长目标**：短视频（3min）/ 中视频（5-8min）/ 长视频（10min+）
- **素材**：是否有已有的研究资料、产品体验笔记、竞品信息

### 2. 搭建骨架

根据类型选择结构模板。详见 [references/structure-templates.md](references/structure-templates.md)。

核心原则：**先展示结果，再讲方法**。永远不要从背景介绍开始。

### 3. 填充内容

用大师的语言风格写作。核心风格约束见 [references/style-guide.md](references/style-guide.md)。

关键风格要素速查：
- 口语化短句，像朋友聊天
- 用「巧了」引出解决方案
- 用「与其...不如...」做对比
- 用「这时候有朋友会问了」预判疑问
- 技术概念必须「说人话」翻译
- 适度自嘲，不端着

签名词汇和固定句式见 [references/vocabulary.md](references/vocabulary.md)。

### 4. 广告植入（仅广告脚本）

广告脚本的植入必须无缝。核心策略：**教程即广告**。
- 先讲通用方法/痛点 → 用「巧了」引出赞助工具 → 用赞助工具演示教程
- 全程保持教学价值，不做硬广
- 语气与非广告脚本完全一致

### 5. 质量检查

完成初稿后，逐项检查：

- [ ] 开场前 10 秒有没有钩子（成果展示 / 对比反差 / 悬念提问）
- [ ] 有没有明确的编号式方法（"今天教你X个方法"）
- [ ] 每个方法结尾有没有小结句（"这就是今天的第X个技巧"）
- [ ] 有没有至少 1 处「巧了」
- [ ] 有没有至少 1 处「与其...不如...」对比
- [ ] 有没有至少 1 处预判观众提问（"这时候有朋友会问了"）
- [ ] 专业术语有没有用大白话翻译
- [ ] 收尾有没有文档引流 + 身份签名
- [ ] 广告脚本：植入是否自然，能否删掉品牌名后仍然是一个完整教程

### 6. 输出格式

默认输出为飞书文档（使用 feishu_create_doc）。脚本正文按页/段落分隔，便于对照录制。

如需分镜标注，在每个段落前用 `【景别】` 标记：`【全景】` `【中景】` `【特写】` `【屏幕录制】`。
