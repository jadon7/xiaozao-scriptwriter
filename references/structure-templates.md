# 脚本结构模板

## 模板一：教程型（最常用）

适用于：AI工具教学、技巧分享、操作指南

```
【钩子 · 5-10秒】
成果展示（"你敢信吗？这些都是我用AI做出来的..."）
或对比反差（"这是你做的AI视频。而这是我修改后的结果。"）

【栏目定位 · 5秒】
"欢迎来到《大师的AI小灶》，这是《AI练习手册》的第X期"
"老规矩，所有案例和提示词我都总结到了文档中"
"话不多说，我们马上开始"

【方法1】
痛点引入 → 解决方案 → 操作演示 → 小结句
"这就是今天分享的第一个技巧：XXX"

【方法2】
"这时候有朋友会问了..." → 解决方案 → 演示 → 小结句

【方法3】（可选，视时长）
进阶技巧 → 演示 → 小结句

【收尾 · 10-15秒】
"以上就是本期视频的全部内容"
"你的点赞收藏是我做视频的最大动力"
"本期视频中出现的所有案例，我也整理到了文档中"
"我是大师，给你单独开小灶，咱们下期再见！"
```

## 模板二：评测型

适用于：工具横评、产品对比、功能测试

```
【钩子 · 5-10秒】
"大评测来了！今天来测评X款热门AI工具"
交代测试规模（"耗时一个月，花费2000多元，测试了200多个案例"）

【评测维度说明】
列出对比维度，说明评测规则（"每组提示词只允许生成三次"）

【逐项对比】（按维度或按案例组织）
每组案例：展示效果 → 逐个点评优缺点 → 横向对比
穿插个人偏好（"我个人更喜欢即梦的效果"）

【总结推荐】
按用户类型给出选择建议：
"如果你是追求极致画质的专业设计师，那么XX会是你的不二之选"
"如果你是AI新手，只想快速体验，XX会是你的好起点"

【收尾】标准收尾
```

## 模板三：观点型

适用于：行业分析、趋势解读、反常识观点

```
【钩子 · 5-10秒】
抛出反常识观点或争议性问题
"你写的90%的提示词，事实上都是无效的"

【论据展开】
用具体案例/数据支撑观点
每个论据后用大白话总结核心逻辑

【反面论证】（可选）
"这时候有朋友要问了，照你这么说XXX就没用了？并不是..."

【金句收尾】
提炼一句可传播的总结句
"原图为主词为辅，顺势不写逆势补。风格数值全删掉，越是简单越靠谱。"

【收尾】标准收尾
```

## 模板四：广告型

适用于：品牌合作、产品推广

```
【钩子 · 5-10秒】
与非广告脚本完全一致的开场方式
绝不以产品名开头

【通用痛点/方法】
先讲一个不依赖赞助产品的通用方法或痛点
让观众先获得价值

【自然过渡】
"巧了，XX正好能解决这个问题"
或 "不得不说，XX在这方面真的很强"

【产品教学】
用赞助产品完成教程演示
保持与非广告脚本完全一致的教学深度和语气

【补充价值】
提供额外的提示词库/案例文档
"这里我总结了XX种常用的..."

【收尾】标准收尾（与非广告完全一致）
```
